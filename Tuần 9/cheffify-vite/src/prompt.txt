Bạn là một Frontend Developer chuyên ReactJS, Vite, React Router và chuyển UI design từ file PDF/Figma/Visily sang code thực tế.

Tôi sẽ cung cấp 2 file thiết kế:
1. visily-multiscreens.pdf là file chính, gồm 9 màn hình cần dựng lại.
2. 3_Week_8_10_ReactJS_UI.pdf là file phụ trợ/tham chiếu để đối chiếu giao diện.

Nhiệm vụ:
Hãy xây dựng một project ReactJS dùng Vite dựa trên 2 file trên. File chính cần bám sát là visily-multiscreens.pdf. Giao diện phải giống bố cục, màu sắc, khoảng cách, card, header, footer, modal, form, dashboard và các section trong file thiết kế nhất có thể.
Yêu cầu kỹ thuật:
- Dùng ReactJS với Vite.
- Dùng React Router DOM để chuyển trang.
- Chia giao diện thành nhiều component rõ ràng, không viết tất cả trong một file.
- Không dùng UI framework như Bootstrap, Ant Design, MUI.
- CSS tự viết bằng file .css hoặc module CSS.
- Code phải clean, dễ đọc, dễ chỉnh sửa.
- Tất cả các button đều phải hoạt động và có tác vụ rõ ràng chứ không phải bấm chỉ hiện ra mỗi cái aleart và không có các button bấm vào những hiển thị trang giống nhau, mỗi button là 1 nhiệm vụ riêng biệt.
- Có thể dùng data mẫu trong file data.js để render card, recipe, comment, dashboard table.
- Ảnh phải chọn đúng nội dung món ăn/công thức/nấu ăn/dashboard, không dùng ảnh bừa. Nếu dùng ảnh online, ưu tiên ảnh từ Unsplash/Pexels hoặc ảnh phù hợp với nội dung từng card.
- Nếu không thể tải ảnh online, tạo thư mục assets và dùng tên ảnh phù hợp để tôi tự thay sau.
- Layout chính cần cố định theo kích thước thiết kế 1440px ở desktop.

Output yêu cầu:
1. Trả về toàn bộ cấu trúc project.
2. Trả về code đầy đủ cho từng file quan trọng.
3. Đảm bảo project chạy được bằng:
   npm install
   npm run dev
4. Nếu có package.json thì dependencies phải dùng registry public npm bình thường, không dùng registry nội bộ.
5. Sau khi code xong, hãy giải thích ngắn gọn:
   - Route nào ứng với trang nào.
   - Component nào dùng chung.
   - Các button đã tương tác như thế nào.
6. Không trả lời chung chung. Hãy tạo code hoàn chỉnh nhất có thể.
